# OpenSpec — cube-registry

Machine-friendly specifications for coding agents. Each spec describes a capability
of the registry. Read these before modifying the schema, CI scripts, or site
generator.

## Structure

```
openspec/
├── specs/
│   ├── entry/     # YAML entry schema; author-provided vs CI-derived fields
│   ├── ci/        # CI pipeline (ownership, quick, slow, health checks)
│   └── site/      # Static site generation (Jinja2 → docs/)
└── changes/       # Active proposals as delta specs
    └── archive/   # Completed changes, prefixed with date
```

The registry is a metadata-only repo: it hosts YAML entries and CI scripts, not
benchmark code. All entries point to PyPI packages.

## For humans

Narrative docs live alongside the README and CONTRIBUTING.md. The published site
at [the-ai-alliance.github.io/cube-registry](https://the-ai-alliance.github.io/cube-registry)
is **generated by CI** from `entries/*.yaml` — never edit `docs/` by hand.
